# Lab Note 001: Hardware Setup

## Why I Acquired Unitree GO2 Air

I self-funded and personally acquired Unitree GO2 Air because I want my AI agent work to move beyond software-only demos and into physical-system research.

Most AI agent examples stay inside chat, browser, or file workflows. A robot introduces a stricter engineering problem: an AI-generated intent must be translated into a safe, validated command boundary before anything physical happens.

## Research Goal

The research goal is to explore how AI Agent workflows can eventually connect with robotics through:

- ROS2 learning
- GO2 hardware familiarity
- Function Calling / Structured Output command schemas
- Human approval boundaries
- Future TRACE Agent Skill drafts
- Public demo evidence after safe testing

## Current Status

- Unitree GO2 Air hardware is owned.
- This repository is a public research log and roadmap.
- ROS2 integration is not completed.
- GO2 connectivity is not claimed as completed.
- No autonomous behavior is claimed.
- No robotics product is shipped.

## Safety Boundary

This repository will not publish:

- Robot serial numbers
- Private location details
- Private network details
- Credentials or API keys
- Unsafe control instructions
- Instructions that encourage unverified physical operation

AI output should be treated as intent, not direct hardware control. Physical execution requires validation, safety checks, and human oversight.
